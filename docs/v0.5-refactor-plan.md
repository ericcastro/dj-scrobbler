# v0.5 Player Refactor Plan

## Goal

DJ Scrobbler v0.5 should own playback directly. Tracklist providers should provide metadata and timing, not act as the playback engine.

The first pass is deliberately YouTube + 1001Tracklists only:

- Always load the app-owned YouTube player for YouTube sets.
- Look up 1001Tracklists in parallel for track metadata, cue times, artwork, IDs, `w/` rows, and mashup/detail rows.
- Drive track highlighting, seeking, resume, and scrobbling from our own playback timer.
- Keep the video movable between the main content area, a sidebar mini player, and an audio-only/hidden-video mode without restarting playback.

## Dormant set79 Work

SoundCloud and set79 are parked during this refactor. The existing source/plugin code and POC scripts are useful future reference for a later bridge:

1. Start from a YouTube set.
2. Find a high-confidence matching SoundCloud URL.
3. Use that SoundCloud URL to resolve a set79 tracklist.
4. Reuse the same normalized timing/highlighting pipeline as 1001Tracklists.

Until that work starts, set79 should stay out of the active source registry and UI so the v0.5 player refactor stays focused.

## Target Contracts

Playback controller:

```js
{
  load(sourceMeta),
  play(),
  pause(),
  toggle(),
  seek(seconds),
  getState() // { isPlaying, currentTime, duration }
}
```

Normalized tracklist:

```js
{
  providerId,
  url,
  tracks: [
    {
      providerTrackId,
      trackNum,
      artist,
      title,
      raw,
      cueSeconds,
      cueDisplay,
      artUrl,
      isId,
      isWWith,
      isMashupComponent,
      noTimestamp
    }
  ]
}
```

## Migration Order

1. Normalize the 1001Tracklists extraction output and stop treating `onclickStr` as the canonical seek handle.
2. Make the YouTube player the normal media surface for all YouTube sets.
3. Move 1001Tracklists loading/extraction behind a metadata-only path.
4. Replace provider DOM now-playing polling with time-based active-track calculation.
5. Collapse `fallback-progress` and `tl-progress` into a single playback progress event.
6. Rework layout modes for full video, mini player, and hidden-video audio playback.

## Current Branch State

- Active registry is YouTube + 1001Tracklists only.
- SoundCloud/set79 modules remain importable under `DORMANT_INTEGRATIONS`.
- YouTube playback now runs through the HTTPS-hosted `https://www.djscrobbler.com/embed/youtube` page.
- 1001Tracklists pages are loaded in a hidden metadata window for extraction.
- Track highlighting and track jumps use `cueSeconds`.
- The footer video-mode button cycles inline, sidebar mini, and audio-only modes.
